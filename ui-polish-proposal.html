Make SimplePass feel alive

The app is feature-complete but static — nothing acknowledges what the user does. This is a working catalog of motion + UI ideas, every demo running live in your real Warm Paper palette. Click any tile to replay. Hover, click, drag mentally — then pick what ships. Nothing here is wired into the app yet.

Design principles

Four rules so motion feels intentional, not decorative. Everything below obeys these.

⚡

React to every act

Tap, send, drop, pair — each gets an instant visual reply under 400ms. Silence reads as "broken".

🌬️

Warm & organic

Spring easing with a touch of overshoot, never robotic linear. Matches the soft clay aesthetic.

🎯

Motion = meaning

Things enter from where they come from. Sent messages pop from the composer; peers ripple in from the network.

🪶

Calm at rest

Idle = subtle breathing only (status halo, scanning radar). No spinning, no noise when nothing's happening.

Cohesion layer — motion tokens

The reason it'll feel "designed" not "decorated": every animation pulls from one shared set of durations and easings. Drop these into :root in styles.css and reuse everywhere.

TokenValueUse for
--dur-1120msHovers, taps, button squish
--dur-2200msState changes, selections
--dur-3320msEntrances, modals, message in
--ease-outcubic-bezier(.22,1,.36,1)Settling into place
--ease-springcubic-bezier(.34,1.56,.64,1)Pops, the "alive" overshoot

All demos here already use these five tokens — that's why nothing feels out of place. Also: everything is wrapped in @media (prefers-reduced-motion: reduce) so motion-sensitive users get instant transitions. Ship that guard from day one.

Suggested rollout

Ordered by impact-per-effort. Phase 1 alone kills the "static" feeling.

1 Quick wins — pure CSS, no logic ~½ day

  • Motion tokens in :root
  • Button squish + tile hover already half-there → standardize
  • Message send/receive entrance animations
  • Online dot breathing halo + shimmer on progress bars
  • prefers-reduced-motion guard

2 Reactive moments — small JS hooks ~1 day

  • Peer pop-in / ripple when onPeersChanged adds an id (diff old vs new list)
  • Staggered grid entrance on mount
  • Spring-select + check badge on tile selection
  • Pairing modal & toast spring-in

3 Delight — the "fun" layer ~1 day

  • Transfer-complete checkmark draw + spark burst
  • Scanning radar empty state
  • Pulsing drag-drop targets with floating hint
  • Optional: subtle sound cue on send / receive (toggle in Settings)

Recommended lib if you want orchestration beyond CSS: framer-motion (now motion) handles list add/remove enter+exit and layout shifts cleanly — ideal for the peer grid and message list. But Phases 1 & most of 2 are doable in plain CSS + a few class toggles, zero new deps.
