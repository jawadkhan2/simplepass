Every element above uses the real component class names from src/styles.css, recolored only by the dark token block — so this is the theme as it would actually ship.

How to ship it

Simplest: replace the :root color vars in src/styles.css with the block below (motion tokens stay as-is). To support a toggle instead, scope these under :root[data-theme="dark"] and set the attribute from Settings.

:root {
  /* Warm Ember — dark theme */
  --paper: #14110e;
  --shell: #100d0b;
  --card:  #1e1a16;
  --line:  #322b24;
  --ink:     #f2e9dc;
  --ink-dim: #a3947f;
  --clay:      #e0764c;
  --clay-soft: #f0a07a;
  --sage:      #9fae74;
  --gold:      #e6b450;
  --surface-2: rgba(255, 255, 255, 0.06);
}

Two spots in styles.css use hardcoded light colors that should also be themed for dark: the progress track #ece3d3 → #2a241e, and the .toast-error light fill #fdf0e8 / text #7a3320 → a translucent clay tint as shown in the mockup. The status-dot offline color #c9bca6 → #4a4036.
